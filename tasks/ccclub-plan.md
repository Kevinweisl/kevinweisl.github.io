## 每篇的標準工作流程

1. **起草**：用 `/write-note` skill 對話式撰寫（Part 2/3 大綱先從對話紀錄撈回來）
2. **入庫預覽**：frontmatter 加 `draft: true` 放進 `content/notes/2026/`——draft 機制已實作，push 也不會公開，本機 `npm run dev` 可預覽
3. **照片**：Kevin 提供原始照片 → Claude 評估插入位置與圖說
4. **發布**：移除 `draft: true`、commit、push main → GitHub Actions 自動部署
5. **導覽**：上一篇/下一篇導覽已自動生成（依日期），無需手動維護

## 照片規格

- 放 `public/images/notes/ccclub/`，檔名用英文小寫加連字號（如 `2016-first-class.jpg`）
- 寬度 1200–1600px、JPG（截圖可用 PNG）
- Markdown 插入後緊接一行 `*斜體圖說*`——網站會自動置中並套用圖說樣式；深色模式自動加邊框
- 文章已有 lightbox：讀者點圖可放大，不用擔心細節看不清
