## 草稿工作流程（repo 是公開的，注意！）

repo 公開 = commit 了就算 `draft: true` 也能在 GitHub 原始碼看到內容。因此未發布文章**不 commit**：

1. **起草**：用 `/write-note` 撰寫，存成 `content/notes/2026/<slug>.draft.md`——`*.draft.md` 已加入 .gitignore，永遠進不了 git
2. **本機審稿**：`npm run dev` 照常渲染（草稿會出現在 Notes 列表，只有本機看得到），慢慢改
3. **照片**：Kevin 提供原始照片 → Claude 評估插入位置與圖說（照片檔發布前也先不 commit）
4. **發布日**：把 `.draft.md` 改名為 `.md` → commit + push main → 自動部署
5. **導覽**：上一篇/下一篇依日期自動生成，無需手動維護

備註：notes pipeline 另支援 frontmatter `draft: true`（隱藏於站上但擋不住 repo 原始碼曝光），適合「已公開過想暫時下架」的情境，不適合未發表草稿。

## 照片規格

- 放 `public/images/notes/ccclub/`，檔名用英文小寫加連字號（如 `2016-first-class.jpg`）
- 寬度 1200–1600px、JPG（截圖可用 PNG）
- Markdown 插入後緊接一行 `*斜體圖說*`——網站會自動置中並套用圖說樣式；深色模式自動加邊框
- 文章已有 lightbox：讀者點圖可放大，不用擔心細節看不清
